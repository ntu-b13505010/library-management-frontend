# Library Management System Frontend

This project is a React + Vite frontend-only mock demo for a Library Management System.

The original real system is Java Swing + JDBC + MySQL. This web frontend does not connect to a backend, database, JDBC code, or MySQL. All current data and behavior are driven by in-memory mock data and mock API functions.

## Tech Stack

- React
- Vite
- React Router
- Axios installed for future backend integration, but not used yet
- Plain CSS

## Install And Run Locally

```bash
npm install
npm run dev
```

Build for production:

```bash
npm run build
```

Preview the production build:

```bash
npm run preview
```

## Current Features

Student side:

- Student login and registration with mock API
- Student dashboard with summary cards
- Search books
- Borrow available books
- View current borrowed books
- Return borrowed books
- View borrow history
- View due reminders
- View book borrow history

Admin side:

- Admin login with mock API
- Admin dashboard with summary cards
- View all borrow records
- Manage users
- Suspend and reactivate users
- Manage books
- Add books
- Remove available books
- Prevent removing borrowed books

## Service Layer

Pages import from `src/services/*Service.js` instead of directly importing mock API modules.

Current flow:

```txt
Page -> Service -> Mock API -> Mock Data
```

This keeps page code easier to preserve later. When backend APIs are ready, the service files can be changed to call real HTTP endpoints while keeping the UI pages mostly unchanged.

## Future Backend Integration Plan

When a Spring Boot backend is available:

1. Keep the page components and routes.
2. Replace internals of the service files with axios requests.
3. Configure a backend base URL in `src/api/httpClient.js`.
4. Map backend response fields to the current frontend shape if needed.
5. Remove or stop using mock data after real endpoints are stable.

## Notes

- This is a frontend prototype only.
- Do not connect directly to MySQL from the frontend.
- Do not modify the Java Swing, JDBC, or database code for this frontend demo.
- The `dist/` folder is generated by `npm run build` and should not be committed unless specifically required.
